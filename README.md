## Furia Chatbot Project ##

This is a project made for the CS Team Furia screening process and it has a prototype chatbot that also uses userdata and allows the team to potencially research the data futher for more improvements and for statistics.

### Features ###

- Chatbot with integrated AI API

- Next games and team roaster

- Statistics to analyse data

- Intuitive graphs to help the team

### How to setup ###

- Download or clone this github repository

- Run setup.py to install required dependencies (must run with permissions to install the said dependencies else it might not work)

- Run run.py and you are good to go!

## Full Documentation ##

This a chatbot project for the CS Team Furia, and the bot is made using react in the frontend and using python on the backend, the language choice is based on the languages used in Furia and the chatbot is developed in a way that it could be scaled to a full chatbot as well.

This chatbot uses OpenRouter API to call the claude API and returns a string, an answer based on the user's input while also using data that is hard-enconded into the chatbot for absolute fact-checking and to avoid that the chatbot speaks confidential information. The chatbot also keeps track of the users topics of interest to use that for futher analytics, and it stores it in a txt file, althrough in an actual implementation it would be recommended to use PostGreSQL to match it with user-id and other user data. The chatbot also contains a screen to analyse the statstics of the users and help the team to find futher insights into what is the trend and are the users mostly researching about, this uses a js library to create intuitive graphs for easier visualization.

## Documentação Completa ##

Este é um projeto de chatbot para o Time CS da Furia, desenvolvido com React no frontend e Python no backend. A escolha das linguagens foi baseada nas utilizadas pela Furia, e o chatbot foi desenvolvido de forma escalável para possivelmente se tornar uma solução completa.

Este chatbot utiliza a API da OpenRouter para chamar a API do Claude e retorna uma resposta baseada no input do usuário, enquanto também usa dados hard-coded para garantir veracidade e evitar que o chatbot divulgue informações confidenciais. O chatbot também monitora os tópicos de interesse dos usuários para análises futuras, armazenando-os em um arquivo .txt. Em uma implementação real, seria recomendado usar PostgreSQL para associar esses dados a IDs de usuário e outras informações. O chatbot também possui uma tela para análise estatística dos usuários, ajudando a equipe a identificar tendências e os assuntos mais pesquisados, utilizando uma biblioteca JavaScript para criar gráficos intuitivos que facilitam a visualização.


#### Chat ####

![スクリーンショット 2025-05-03 185752](https://github.com/user-attachments/assets/06e85a03-cb63-4200-885f-0746069775f5)

O chat é a parte principal do chatbot, alimentado pelo backend em chatbot.py, que chama o Claude através da API da OpenRouter. Ele compara o input do usuário com o banco de dados interno do chatbot.py (ou conjunto de dados) e responde com informações precisas sobre a pergunta. Este chat pode ser melhorado com uma API de IA paga e carregando dados dinamicamente de um banco de dados, por exemplo, mas como se trata de um protótipo, tudo é gerenciado dentro do chatbot.py por simplicidade.

O chatbot responde a maioria das perguntas básicas, mas pode ter dificuldades com perguntas muito complexas ou longas (limite do token gratuito da API), então isso é algo a se considerar.

#### Análises ####

![スクリーンショット 2025-05-03 185921](https://github.com/user-attachments/assets/2a3c6552-66af-4232-9525-3ed1a73ec296)

A página de análises contém todas as informações que a equipe precisaria para melhorar vendas e analisar o conteúdo do chatbot: o que os usuários mais perguntam e quais são os principais tópicos. Para este protótipo, os dados foram gerados aleatoriamente por IA, pois seria muito custoso implantar um servidor completo usando Docker apenas para um protótipo, além de exigir múltiplas interações com a API da OpenRouter, que atualmente estão limitadas (plano gratuito). Portanto, optou-se por simular os dados.

A ideia é usar os dados enviados pelo chatbot para alimentar as tabelas e utilizar gráficos para melhor visualização, tornando a interface mais amigável. Futuramente, poderiam ser adicionados recursos como exportação direta para Excel ou PowerBI para relatórios precisos, ajudando as equipes de gestão.

#### A fazer ####

Este é um protótipo e ainda há funcionalidades a serem adicionadas e ajustes a serem feitos. Para facilitar, listei-os abaixo:

- Suporte para mobile: o site foi feito em React, mas o CSS não está otimizado para dispositivos móveis, podendo haver problemas de compatibilidade.

- API da OpenRouter: o token da API da OpenRouter precisa ser renovado a cada poucas horas, então o programa não pode ser implantado independentemente. Para mais informações, consulte [OpenRouter API](https://openrouter.ai/docs/quickstart).

- Integração com um banco de dados PostgreSQL para coletar e atualizar dados específicos dos usuários.

- Adicionar suporte a emojis no chat e na página para corrigir os ícones de bandeiras, por exemplo.
